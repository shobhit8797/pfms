# 50:30:20 Budgeting Tracker

A parallel surface to the legacy expense/account features, porting the
spreadsheet's 50:30:20 model with low-friction capture, receipt attachments,
AI statement import (OpenRouter), analytics, and an iOS-ready REST + sync API.

## Data model (prisma/schema.prisma)
`BudgetProfile` (income split + weekly limit, one active per user, history kept),
`Category` (NEED|WANT|SAVING), `PaymentMode`, `Transaction` (the ledger; `type`
denormalized from category, soft-deletable), `Receipt`, `StatementImport`,
`StagedTransaction` (review gate — never in the ledger until committed),
`ApiToken` (bearer auth for iOS). All userId-scoped with `@@index([userId, updatedAt])`
for delta sync; soft-deletable models carry `deletedAt`.

## Architecture
Business logic lives in `lib/services/*` as plain `(userId, input)` functions that
throw typed `ServiceError`s (`lib/errors.ts`). Two adapters wrap them:
- **Server Actions** (`app/actions/budget/*`) for the web UI — `auth()` → Zod
  (`lib/validation/budget.ts`) → service → `revalidatePath`.
- **REST handlers** (`app/api/v1/**`) for iOS — `requireApiUser` (`lib/api-auth.ts`,
  session cookie OR bearer token) → service → JSON. `withApiUser` maps errors to HTTP.

## Key flows
- **Onboarding**: `/dashboard/budget/setup` → income → 50:30:20 split + seeds
  defaults (`lib/services/seed.service.ts`).
- **Ledger**: `/dashboard/budget/transactions` — quick add/edit/soft-delete, type
  auto-fills from category, receipt attach (Vercel Blob client upload via
  `/api/v1/blob/upload` → `linkReceipt`).
- **Statement import**: `/dashboard/budget/imports` → upload → `after()` runs
  `runExtractionForImport` (CSV via `lib/statement-parser`; PDF→pdf-parse text;
  IMAGE→vision) → OpenRouter structured JSON (`lib/llm/*`, validate + retry-once-then-FAIL)
  → enrich (validate suggested ids, duplicate guess) → staged review table → commit
  approved rows with shared `importBatchId` → undo batch. Cron sweep
  (`/api/v1/internal/extract-sweep`, `vercel.json`) re-drives stuck imports.
- **Analytics**: `lib/analytics/calc.ts` (pure, testable) + `analytics.service.ts` —
  50:30:20 budget-vs-actual (prorated), weekly ratio, monthly daily-spend + 0–10
  adherence score, yearly heatmap.
- **Sync**: `GET /api/v1/sync?since=` returns `{upserts, deletes}` per entity.

## Env vars
`OPENROUTER_API_KEY`, `OPENROUTER_MODEL` (default `openai/gpt-4o-mini`),
`OPENROUTER_VISION_MODEL` (default `openai/gpt-4o`), `BLOB_READ_WRITE_TOKEN`
(Vercel Blob), `EXTRACT_WORKER_SECRET`, `CRON_SECRET`.

## Notes / follow-ups
- Migration `prisma/migrations/*_budget_5030_20` is additive (no DROPs); apply with
  `prisma migrate deploy`.
- Blobs use `access: "public"` (unguessable URLs) in v1; private + signed reads is a
  follow-up once iOS needs authenticated reads.
- Adherence score uses the PRD v1 default; the exact spreadsheet weighting is an open
  question (see PRD §13).
